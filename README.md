# Design Patterns in Java - Interactive Learning Repository

This project is a practical laboratory for implementing and understanding various software design patterns in Java.

## 🚀 About the Project

The main goal of this repository is to serve as an active learning environment. It is designed to practice identifying code smells, tightly coupled logic, and poor architectural decisions, and subsequently refactoring them into robust, clean, and highly scalable architectures using SOLID principles and standard Design Patterns.

## 🤖 AI-Assisted Learning Workflow

This project utilizes a unique, AI-assisted workflow for continuous practice:

1. **Problem Generation (AI Assisted)**: A custom AI Skill is used to automatically generate a messy, tightly-coupled codebase representing a specific real-world scenario (the "problem" package).
2. **Refactoring (Manual)**: The developer manually studies the problem and implements the correct design pattern solution in a `refactor` package, entirely without AI assistance. This guarantees active learning and deep understanding of the pattern's mechanics.

## 🛠 Tech Stack

* **Java 17**: Core language used for all pattern implementations.
* **Maven**: Dependency and build management.
* **Spring Framework (Context 6.1.4)**: Used in specific patterns to demonstrate real-world Dependency Injection, Registry patterns, and Inversion of Control.

## 📂 Project Structure

The codebase is organized by design pattern and divided into two main stages for each exercise:

* `com.best.practices.[pattern_name].problem`: Contains the initial, flawed implementation generated by the AI.
* `com.best.practices.[pattern_name].refactor`: Contains the clean, manually refactored solution applying the specific design pattern.

## 🌟 Learning Objectives

* Master creational, structural, and behavioral design patterns.
* Practice transforming legacy or tightly coupled code into modular architectures.
* Learn to integrate raw design patterns with modern frameworks like Spring.
* Enforce Strong Typing, the Open/Closed Principle (OCP), and separation of concerns.
